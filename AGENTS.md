# AI Agent Instructions for bartplasmeijer.nl Blog

This is a [Hugo](https://gohugo.io/) static site generator blog about AI, architecture, and engineering hosted on GitHub Pages.

## Quick Start

**Build**: `hugo` (outputs to `./public/`)  
**Serve locally with drafts**: `hugo server -D` (available at `http://localhost:1313`)  
**Serve production only**: `hugo server` (ignores drafts)

Use the VS Code tasks if available:
- "Serve Drafts" (`hugo server -D`)
- "Build" (`hugo`)

## Project Structure

```
content/
  about.md           # Author bio and professional summary
  posts/             # Blog posts with YYYY-MM-DD- prefix and .md extension
config.toml          # Hugo configuration (baseURL, theme, pagination, social links)
themes/archie/       # Active theme (minimal, fast)
themes/ananke/       # Backup/alternative theme
static/              # Static assets (images, etc.)
public/              # Build output (generated by `hugo`, commit to gh-pages branch)
```

## Content Conventions

### Blog Posts

- **Location**: `content/posts/`
- **Naming**: `YYYY-MM-DD-<slug>.md` (e.g., `2026-04-10-my-post.md`)
- **Front Matter** (TOML):
  ```toml
  +++
  title = "Post Title"
  date = "2026-04-10"
  draft = false              # Set to true to exclude from production builds
  type = "post"
  tags = ["tag1", "tag2"]    # Used for categorization
  linkedin_url = "..."       # Optional: LinkedIn post URL
  +++
  ```
- **Content**: Markdown with Hugo shortcodes; MathJax and KaTeX enabled for equations

### Key Front Matter Fields

- `draft = true` prevents publication unless using `hugo server -D`
- `tags` are required for content organization (lowercase, descriptive)
- `linkedin_url` is used by LinkedIn dispatch workflows

## Hugo Configuration

See [config.toml](config.toml) for:
- `baseURL`: https://www.bartplasmeijer.nl/
- `theme`: "archie" (active), "ananke" (available)
- `pagination.pagerSize`: 3 articles per page
- `params.mathjax` & `params.katex`: Both enabled for math rendering
- Social links (GitHub, Twitter) defined in `[[params.social]]`
- Main menu items in `[[menu.main]]`

## Deployment

GitHub Actions workflow ([.github/workflows/hugo.yaml](.github/workflows/hugo.yaml)):
- **Trigger**: Pushes to `main` branch
- **Build env**: Hugo 0.111.3, Dart Sass Embedded
- **Output**: Deployed to GitHub Pages at https://www.bartplasmeijer.nl/
- **Submodules**: Uses theme submodules (fetch with `--recursive`)

## Common Tasks

### Create a New Post
1. Create `content/posts/YYYY-MM-DD-<slug>.md`
2. Use front matter template from existing posts
3. Write content in Markdown
4. Set `draft = false` when ready to publish
5. Run `hugo server -D` to preview locally
6. Push to `main` to deploy

### Preview Changes
- **With drafts**: `hugo server -D` (includes draft posts)
- **Production**: `hugo server` (excludes drafts)
- **Build for testing**: `hugo` (outputs to `./public/`)

### Theme Customization
- Active theme: `themes/archie/` (see theme's [README.md](themes/archie/README.md) and [config.yaml](themes/archie/config.yaml))
- Alternative: `themes/ananke/` (see theme's [README.md](themes/ananke/README.md))
- Layouts, assets, and static files defined in theme directories

## Important Notes

- **Submodules**: Themes are Git submodules; use `git clone --recursive` or `git submodule update --init --recursive`
- **Build output**: `public/` is generated; never edit files in `public/` directly
- **Image assets**: Store in `static/images/` and reference as `/images/...` in content
- **Dependencies**: Node.js optional; `npm ci` runs only if lock files exist
- **Hugo version**: 0.111.3 (matches CI environment)
